# Annotation Notes

## Annotation philosophy
Prefer operational correctness over stylistic preference.

## Important annotation categories
- item identity
- project attribution
- sample attribution
- event type
- condition slots
- issue / blocker presence
- follow-up necessity
- evidence support

## Follow-up annotation
Annotate intent category, not exact wording.

Examples:
- ask_project_identity
- ask_sample_identity
- ask_temperature
- ask_duration
- ask_attachment_context
- ask_repeatability

## Ambiguity rule
If the input is genuinely ambiguous, do not force a single gold interpretation.

# 项目当前状态（中文简述）

## 当前定位
ChemClaw-Lab 目前被定义为：

**面向化学实验室日常运行的、消息原生的、事件驱动的多 Agent 智能助理框架**

它关注的不是泛化聊天，而是三类高频实验室操作流：

- 化学品与耗材管理
- 实验记录与项目记忆整理
- 计划制定与组内仪器预约

## 已经明确下来的关键点
- 目标实验室规模：约 10 人
- 当前已知数据底座：化学品 Excel
- 仪器预约范围：先做组内仪器
- 当前目标：论文 demo + benchmark + 开源框架
- 项目偏向：科研智能助理，而不是普通秘书类 bot

## 已完成的设计内容
- 系统整体架构
- Orchestrator + A/B/C + Audit 设计
- event-centric memory 核心思想
- 数据库 schema 草案
- benchmark 草案
- agent prompt 正式版草案
- 论文大纲与前几章初稿
- 面试 1 页 PPT 叙事与示意图提示词

## 下一步最重要的事
1. 用合成数据先把系统跑起来  
2. 尽快落地 benchmark 样例  
3. 让 Agent A / B / C 跑通最小闭环  
4. 开始积累可展示的 demo case  
5. 逐步补真实实验室数据和案例

## 当前建议
不要一开始就追求“全自动实验室 AI 平台”。  
先把“小型化学实验室的消息原生操作助手”做扎实，这样论文和开源都会更稳。
